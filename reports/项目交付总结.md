# doc-textify 项目交付总结

## 项目定位

`doc-textify` 的目标是成为文本大模型的低成本“眼睛”：先把图片、PDF、截图、扫描件和部分图表编译成结构化文本，再交给没有视觉能力的大模型阅读。

它不是文生图复现工具。文生图复现很难作为客观评测，因为生成模型会自行补全、改写和风格化。项目采用更可验证的路线：输出 Markdown/TXT/LLM 文本和 JSON sidecar，再用人工标注的 expected JSON 做结构化评分。

## 核心能力

- 数字 PDF 原生文本抽取。
- 扫描 PDF 渲染后 OCR。
- 图片 OCR，支持中文/英文混排。
- 页面块模型：`Document -> Page -> Block`。
- 输出 Markdown、TXT、JSON 和紧凑 LLM 文本协议。
- 基础版面分析：标题、段落、列表、页眉页脚、阅读顺序。
- 图表解析：面板、轴标签、红色区间线、散点、结构化 `chart_data`。
- 独立评测框架：比较实际 JSON 与人工 expected JSON。

## 低 Token LLM 协议

新增 `--format llm` 和 `--format all`。LLM 协议输出会把图表数据压成类似下面的文本：

```text
DOC_TEXTIFY_LLM_PROTOCOL v1
source: 对比图.jpg
pages: 1

[page 1 size=1280x1971]
chart_data:
  panel a:
    intervals: class 0 depth 0.0-5.4; class 1 depth 5.0-7.2
    points: class 1 depth 4.0; class 2 depth 0.5
```

这比完整 JSON 更省 token，也比普通 OCR 文本更适合大模型推理。

## 仍需继续推进

要更接近“真正的大模型眼睛”，下一阶段应集中优化图表坐标校准：

1. 从图表网格线和 OCR 刻度建立 y 像素到深度值的映射。
2. 用映射重算红色竖线区间和散点深度。
3. 区分真实数据点、线段端点、图例标记和印刷噪声。
4. 增加更多样本：表格、公式、倾斜拍照、手机截图、多栏论文。
5. 增加下游问答评测：让文本大模型只读 `.llm.txt` 回答图像相关问题。
