## 2. 本轮优化目标

本轮目标是继续推进“让文本大模型用低 Token 成本读取图片/PDF 内容”：

1. 收紧 OCR/PDF 文本块分类，减少普通正文被误判为 heading。
2. 改进图表数据提取，输出可评测的结构化 chart_data。
3. 新增紧凑 LLM 文本协议，方便直接喂给没有视觉能力的大模型。
4. 为图表读数加入显式误差范围，避免伪装成不可靠的精确值。
5. 清理项目结构，集中报告文件，准备上交 GitHub。

## 3. 已完成的关键改动

### 3.1 Tesseract 自动发现

当前系统中 Tesseract 不在 PATH，但存在于：

D:\python\auto_monitor\asd\tesseract.exe

项目已加入自动发现逻辑，可在该路径找到 Tesseract，并使用其内置 chi_sim 与 eng 语言包。

### 3.2 OCR 文本规范化

新增中文 OCR 后处理：

- 合并中文字符之间的异常空格。
- 将 /m 深度、深度 /m 规范为 深度/m。
- 将 真实 类别 规范为 真实类别。
- 将 预测 类别 规范为 预测类别。
- 将常见误识别 钼孔 修正为 钻孔。

### 3.3 图表不确定度表达

图表读数现在不是只输出一个裸数值，而是输出带误差范围的视觉测量结果：

~~~text
class 0 depth 0.0-5.4 +/- 0.7
class 2 depth 0.5 +/- 0.56
~~~

这更适合给文本大模型使用：模型可以知道图中表达的大致事实，也知道该事实不是无限精确。

### 3.4 LLM 低 Token 输出

新增：

- --format llm：输出 .llm.txt 和 JSON。
- --format all：输出 Markdown、TXT、LLM 文本和 JSON。

LLM 文本协议示例：

~~~text
DOC_TEXTIFY_LLM_PROTOCOL v1
source: 对比图.jpg
pages: 1

[page 1 size=1280x1971]
chart_data:
  panel a:
    intervals: class 0 depth 0.0-5.4 +/- 0.7; class 1 depth 5.0-7.2 +/- 0.7
    points: class 1 depth 4.0 +/- 0.56; class 2 depth 0.5 +/- 0.56
~~~

## 7. 当前能力边界

已经可靠的部分：

- 数字 PDF 原生文字抽取。
- 扫描 PDF 渲染后 OCR。
- 图片 OCR，包含中文和英文混合。
- Markdown/TXT/LLM/JSON 输出。
- 图像中关键术语、面板、caption、坐标轴标签提取。
- 可输出带误差范围的结构化 chart_data。

仍需优化：

- 图表深度坐标已加入显式误差范围；下一步应基于水平网格线和刻度 OCR 进一步缩小误差。
- 红色散点需要更强噪声过滤。
- PDF 标题分类仍偏多，后续应结合字号、位置、目录结构和段落长度进一步收敛。
- 复杂表格、公式和图片说明还需要独立评测集。

## 8. 下一步建议

为了把图表读数从“带误差可用”推进到“更高精度可用”，下一轮建议集中做图表坐标校准器：

1. 检测每个面板的水平网格线。
2. OCR 左侧深度刻度，例如 0、2、4、6。
3. 建立 y 像素到深度值的线性映射。
4. 用该映射重算 interval 和 point 的深度。
5. 对红点做端点剔除，减少误检。
